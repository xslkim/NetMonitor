# 输入文档写作规范

> 本文档定义口播稿和视觉素材指南的格式要求。
> 遵循这些规范可以确保自动化视频制作流程正确解析和渲染。

---

## 一、口播稿（script.md）

### 1.1 整体结构

```markdown
# 视频标题

---

第一段口播正文...

---

第二段口播正文...

---

*可选尾注，不纳入视频*
```

**规则：**
- 第一行必须是 `# 标题`，自动提取为视频标题和片尾信息
- 用 `---`（三个以上短横线，前后各空一行）分隔段落
- 第一个 `---` 之前的内容为标题区，不作为口播内容
- 最后一个 `---` 之后如果仅有斜体文字（`*...*`），视为尾注，自动丢弃
- 每个 `---` 之间的内容是一个**段落**（对应视频中的一个场景）

### 1.2 字幕控制

口播文字会直接转为 TTS 语音和屏幕字幕。**字幕过长会溢出屏幕或自动换行导致排版混乱**，因此必须控制每行长度。

#### 字幕行长度上限

| 画面比例 | 屏幕宽度 | 字幕字号 | 每行上限 | 说明 |
|---------|---------|---------|---------|------|
| 16:9 横屏 | 1920px | 36px | **20 个中文字符** | B站/YouTube 标准 |
| 9:16 竖屏 | 1080px | 32px | **14 个中文字符** | 抖音/小红书 |
| 1:1 方屏 | 1080px | 34px | **16 个中文字符** | Instagram |

> 英文和数字按 0.5 个中文字符计。标点符号按 1 个字符计。

#### 用空行分割字幕块

每个空行分隔一个**字幕块**。每个字幕块在屏幕上独立显示，对应一次字幕刷新。

```markdown
每个月看到云服务器的账单，我都肉疼。

我有几台 Windows 服务器跑在云上，
按流量计费。
```

上面有 2 个字幕块：
- 块 1："每个月看到云服务器的账单，我都肉疼。"（一行显示）
- 块 2："我有几台 Windows 服务器跑在云上，按流量计费。"（TTS 连读，字幕两行显示）

#### 用换行控制字幕行数

字幕块内的**硬换行**（行末直接回车）会被保留为字幕内的行分割：

```markdown
我有几台 Windows 服务器跑在云上，
按流量计费。
```

渲染效果：
```
我有几台 Windows 服务器跑在云上，
按流量计费。
```

**每个字幕块最多 2 行。** 超过 2 行的块自动截断第 3 行及以后到下一个字幕块。

#### 字幕块 vs TTS 朗读

- 空行：字幕切换 + TTS 短暂停顿（~0.3 秒）
- 硬换行：仅影响字幕显示分行，TTS 连读不停顿
- 句号/问号：TTS 自然停顿，但字幕不切换

### 1.3 素材切换标记

默认行为：一个段落（`---` 之间）的所有素材按时间均分。
如果需要精确控制口播到哪句话切换画面，使用**素材切换标记**：

```markdown
>>> 云账单截图

每个月看到云服务器的账单，我都肉疼。

>>> 任务管理器动画

我有几台 Windows 服务器跑在云上，
按流量计费。
每 GB 出站流量几毛钱，听起来不多，
但架不住有些程序特别能吃。
```

**语法：** `>>>` 开头的行是素材切换标记

- 标记文字应与素材指南中的素材标题**对应**（不需要完全一致，模糊匹配）
- 标记行不会被 TTS 朗读，也不会出现在字幕中
- 标记行之后到下一个标记行（或段落结束）之间的文字，对应该素材的展示时间
- 如果没有 `>>>` 标记，该段落内的素材按描述文字长度成比例分配时间

### 1.4 语速与停顿

```markdown
每个月看到云服务器的账单，我都肉疼。

（停顿）

我有几台 Windows 服务器跑在云上，
按流量计费。
```

| 标记 | 效果 |
|------|------|
| 空行 | TTS 短暂停顿 ~0.3s + 字幕切换 |
| `（停顿）` 或 `（pause）` | 额外 1 秒静音（不显示字幕） |
| `（长停顿）` | 额外 2 秒静音 |
| 空行 × 2 | 等价于空行 + `（停顿）` |

### 1.5 重音与高亮

```markdown
限速要用 **WinDivert**——一个开源的网络数据包拦截库。
```

- `**加粗**` 的词在字幕中用强调色（`textAccent`）高亮显示
- 不影响 TTS 朗读

### 1.6 完整示例

```markdown
# 我用 AI 两小时造了一个限速工具

---

每个月看到云服务器的账单，我都肉疼。

>>> 云账单截图

我有几台 Windows 服务器跑在云上，
按流量计费。
每 GB 出站流量几毛钱，听起来不多，
但架不住有些程序特别能吃。

>>> 任务管理器动画

Windows Update 后台狂拉补丁、
日志同步服务一股脑往外推数据、
某个写得不太好的爬虫程序跑起来没有节制。
月底一看账单，光流量费就好几百。

---

>>> 工具定价对比

这需求不复杂，市面上有现成工具。
NetLimiter、GlassWire、SoftPerfect……
我挨个看了一遍，然后我看到了价格。

>>> 黑盒金句

花钱买一个黑盒来省钱，
这逻辑说不通。

（停顿）

我决定自己造一个。

---

*全文约 2000 字，预计录制 10 分钟。*
```

## 二、视觉素材指南（visual-guide.md）

### 2.1 整体结构

```markdown
# 视频标题配套素材指南

---

## 第一段：段落标题

**[类型]** 素材标题
素材详细描述...

**[类型]** 素材标题
素材详细描述...

---

## 第二段：段落标题

...
```

**规则：**
- 用 `---` 分隔，与口播稿的段落**一一对应**（第 N 个分隔块对应口播稿第 N 段）
- 每个 `## 二级标题` 标注段落名称（与口播稿段落的主题对应）
- 每个 `**[类型]**` 开头的块定义一个素材

### 2.2 素材类型

| 类型标记 | 含义 | Agent 生成方式 |
|---------|------|---------------|
| `[动画]` | 有时间轴的动态画面 | React 组件 + Remotion 动画 API |
| `[截图]` | 模拟静态截图 | React 组件 + 缓慢缩放(Ken Burns) |
| `[录屏]` | 模拟软件操作过程 | React 组件模拟 UI + 逐步动画 |
| `[文字卡]` | 纯文字画面 | 通用 TextCard 组件 |
| `[代码]` | 代码展示/打字效果 | 代码高亮组件 + 逐行出现 |

### 2.3 素材描述怎么写

**描述越具体，AI 生成的组件越准确。** 好的描述包含：

#### 必写项

1. **画面内容**：画面上有什么元素
2. **数据/文案**：具体显示的文字、数字、名称

#### 选写项（写了效果更好）

3. **动画流程**：元素出现的先后顺序、运动方式
4. **时间轴**：关键帧的时间点
5. **样式要求**：颜色、字号、布局等特殊要求
6. **代码块**：需要原样显示的内容用 ` ``` ` 包裹

#### 描述质量对比

**差（太模糊）：**
```markdown
**[动画]** 展示三个工具的价格。
```

**好（具体可执行）：**
```markdown
**[截图]** 工具定价对比
三列等宽卡片横向排列，每个卡片包含：
- NetLimiter：$29.95 / 台
- GlassWire：$49 / 年
- SoftPerfect Bandwidth Manager：$49 / 台
每个价格用红色大字标注，卡片用红色边框。
卡片从左到右依次淡入，间隔 0.5 秒。
```

### 2.4 文字卡的特殊写法

文字卡的**精确文案**必须用引号或代码块标出：

```markdown
**[文字卡]** 黑盒金句
画面中央显示：
> 「花钱买一个黑盒来省钱，这逻辑说不通。」

深色背景，文字金色，居中，淡入停留 2 秒后淡出。
```

### 2.5 代码展示素材

如果素材涉及展示代码，指定代码来源：

```markdown
**[代码]** 令牌桶代码快进
使用代码文件：TokenBucket.h
代码行逐行从上到下出现，每帧 2-3 行。
深色编辑器风格（VS Code Dark+），
行末有闪烁光标。3-5 倍速播放效果。
```

- `使用代码文件：xxx` 指定 `src/data/source-samples/` 中的文件名
- 如果不指定文件，也可以直接在 ` ``` ` 代码块里写内容

### 2.6 动画时间轴写法

对于复杂动画，建议写明时间轴：

```markdown
**[动画]** 令牌桶原理图
- 0-2s：桶满，3 个数据包到达并消耗水位，绿色放行
- 2-4s：水位降到 0，新数据包被阻断（红色 ×）
- 4-6s：水龙头继续滴水，水位缓慢上升
- 6-8s：水位够了，被阻断的包重新放行
```

Agent 会根据这些时间点生成对应的 `interpolate()` 关键帧。

### 2.7 素材标题与口播稿 `>>>` 标记的对应

口播稿中的 `>>> 云账单截图` 会与素材指南中最近匹配的素材标题绑定：

| 口播稿 `>>>` 标记 | 匹配规则 |
|---|---|
| `>>> 云账单截图` | 匹配素材标题含"云账单"或"账单"的 |
| `>>> 三层架构动画` | 匹配素材标题含"架构"的 |
| `>>> 素材 7` | 匹配第 7 个素材（按编号） |

匹配规则：
1. 优先精确匹配标题
2. 其次关键词模糊匹配
3. 最后按 `素材 N` 格式按编号匹配
4. 无法匹配时使用段落内的顺序位置

### 2.8 片尾段

最后一段建议为片尾信息：

```markdown
## 片尾

**[文字卡]** 片尾信息
黑底白字：
- 项目名称
- 技术栈关键词
- 源码链接（如有）
- 「感谢收看」
```

### 2.9 完整示例

```markdown
# 素材指南 — 我用 AI 两小时造了一个限速工具

---

## 第一段：流量费用的烦恼

**[截图]** 云账单截图
模拟云服务商控制台费用页面：
- 深色面板，标题「流量费用明细 — 2025年3月」
- 表格：出站流量 580GB / ¥0.80/GB / 合计 ¥464
- 「合计: ¥580」用红色大字高亮
- 缓慢放大效果

**[动画]** 任务管理器动画
仿 Windows 11 任务管理器网络标签页：
- 列：进程名 | PID | 网络使用
- 3 行数据动态更新：
  - Windows Update：20 MB/s
  - LogSync.exe：5 MB/s
  - crawler.py：8 MB/s
- 每行逐个黄色高亮，底部滑入：「月底流量费: ¥580」

---

## 第二段：现成工具太贵了

**[截图]** 工具定价对比
三个等宽卡片横向排列：
- NetLimiter：$29.95 / 台
- GlassWire：$49 / 年
- SoftPerfect：$49 / 台
价格红色大字，红色边框，卡片依次淡入。

**[文字卡]** 黑盒金句
> 「花钱买一个黑盒来省钱，这逻辑说不通。」

金色文字，深色背景，居中，停留 2.5 秒。

---

## 片尾

**[文字卡]** 片尾信息
> MyProject
> C++ / Python / React
> 感谢收看

黑底白字，4 秒。
```

## 三、两份文档的关系

```
口播稿                          素材指南
======                          ========
# 标题                          # 标题配套素材
---                             ---
段落 1 文字                     ## 段落 1
  >>> 素材切换标记  ←──对应──→    [类型] 素材 A 描述
  >>> 素材切换标记  ←──对应──→    [类型] 素材 B 描述
---                             ---
段落 2 文字                     ## 段落 2
  >>> 素材切换标记  ←──对应──→    [类型] 素材 C 描述
---                             ---
...                             ...
```

**核心规则：**
1. 两份文档的 `---` 段落数量必须一致
2. 段落 N 内的素材切换标记，对应素材指南第 N 段内的素材
3. 如果口播稿无 `>>>` 标记，段落内素材按描述长度成比例分配时间
4. 如果口播稿有 `>>>` 标记，每个标记到下一个标记（或段落结束）之间的朗读时长即为该素材的展示时长

## 四、自查清单

写完两份文档后，按以下清单检查：

### 口播稿

- [ ] 第一行是 `# 标题`
- [ ] 段落之间用 `---`（前后空行）分隔
- [ ] 每个字幕块（空行之间的文字）不超过 2 行
- [ ] 每行字幕长度不超过目标比例的上限（16:9 → 20字，9:16 → 14字）
- [ ] `>>>` 标记的数量与素材指南中对应段落的素材数量一致（如果使用了标记）
- [ ] 没有孤立的 `>>>` 标记（标记后面必须有口播文字）
- [ ] 总字数在预期范围（中文 ~200字/分钟，估算视频时长）

### 素材指南

- [ ] 段落数与口播稿一致
- [ ] 每个素材都有 `**[类型]**` 标记
- [ ] 每个素材都有标题（标记后同行的文字）
- [ ] 文字卡的精确文案用 `>` 引用块标出
- [ ] 代码展示素材指定了代码文件名或提供了代码块
- [ ] 复杂动画有时间轴描述
- [ ] 最后一段包含片尾文字卡
